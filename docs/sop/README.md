# Standard Operating Procedures

Standard operating procedures (SOPs) capture mandatory policies that keep 3leaps projects consistent. Unlike guides (how-to) or standards (specifications), SOPs are prescriptive: "this is the policy, follow it."

## Available SOPs

| Document                                 | Purpose                                        |
| ---------------------------------------- | ---------------------------------------------- |
| [Stream Output Policy](stream-output.md) | stdout/stderr discipline for tool chainability |

## When to Add a New SOP

Create an SOP when:

- A policy spans multiple repos and we need consistent enforcement
- The policy has clear compliance criteria that can be tested
- Violations cause real problems (wasted time, broken pipelines, user confusion)
