# Release v0.1.21

**Signed publication-policy attestation.**

## Overview

v0.1.21 updates release mechanics and their documentation. No schema, contract,
capability token, or engineering-principle obligation changes.

## Changes

- `make release-tag` validates the complete version-tag ruleset before creating
  a tag.
- Signed annotated tags carry a canonical SHA-256 fingerprint of the expected
  publication policy.
- Release CI validates the read-only ruleset view and requires the signed policy
  fingerprint after pinned-key signature verification.
- Release-control tests cover complete and read-only policy validation plus
  missing or incorrect tag attestations.
- PDR-0004 and `RELEASE_CHECKLIST.md` describe the release gate and its operator
  checks.
