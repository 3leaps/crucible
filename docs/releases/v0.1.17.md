# Release v0.1.17

**Baseline release, data artifact metadata hardening, and repository guidance alignment.**

## Overview

v0.1.17 tightens the `data-artifact/v0` contract for self-describing columnar
formats, records the physical-file-metadata concern as resolved into the
Metadata Is Content rule, and adds a deliberate public reference-producer
citation. It also aligns repository guidance, attribution docs, release history,
and examples for public standards use.

## Highlights

### Data Artifact Physical Metadata Is Content

Boundary-crossing representations now treat embedded physical file metadata as
content. For fields whose sensitivity is `controlled` or `restricted`, or whose
protection tags include `direct_identifier`, `linkage_key`, or
`source_structure`, producers must suppress or make opaque per-column statistics
such as row-group or page min/max values and page-index bounds.

Membership-oracle structures, including per-column Bloom filters, must be
omitted for those fields. Producer profiles state how their concrete format
satisfies the requirement, with writer-side omission preferred over export-time
rewrite.

### Data Artifact Validator Coverage

The validator checklist now includes physical metadata suppression for restricted
class columns in boundary-crossing columnar representations. The same checklist
retains the fail-closed verify-not-trust line for small-cell suppression:
conforming gates verify emitted aggregates rather than accepting a producer's
self-asserted suppression flag.

### Review-Loop Fold

The physical-file-metadata review-loop item is now folded into Metadata Is
Content. Format-specific suppression mechanics remain producer-profile details,
not base-contract requirements.

### Reference Producer Citation

The examples intentionally cite
[`fulmenhq/sumpter`](https://github.com/fulmenhq/sumpter) as a public reference
producer for the portable contract. The citation is non-normative: concrete
producer schemas remain downstream-owned unless shared-profile convergence
justifies promotion.

### Public Repository Guidance

Repository-facing guidance is aligned for public standards use:

- root `AGENTS.md` is a concise contributor-agent exemplar;
- AI attribution guidance is generic and public-facing;
- docs and examples use adopting-repository language;
- the obsolete repository safety playbook is removed from the baseline.

### Portable Schema Framing

ADR-0002 is retitled as a portable schema contract, and direct references are
updated across the decisions index, docs index, changelog, release history,
ADR-0003, and PDR-0001.

## Migration Notes

- **Columnar producers:** suppress or make opaque restricted-class physical
  metadata before crossing a representation boundary. Prefer writer-side
  omission over export-time rewrite.
- **Producer profiles:** document how the concrete format satisfies physical
  metadata suppression. Keep format-specific mechanics in the profile rather
  than the base contract.
- **Validators:** fail closed when boundary-crossing columnar representations do
  not declare physical metadata suppression for restricted-class columns.
- **Reference producer users:** treat the `fulmenhq/sumpter` citation as a public example,
  not a normative schema dependency.
